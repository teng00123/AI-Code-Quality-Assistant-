# 简单质量评分算法 - Phase 1 MVP

## 🎯 算法概述

这是Phase 1 MVP中实现的基础代码质量评分算法，采用**四维度加权评分模型**：

- **可维护性** (30%) - 代码结构的易修改程度
- **可读性** (25%) - 代码的清晰易懂程度  
- **安全性** (25%) - 代码的安全风险程度
- **性能** (20%) - 代码的执行效率潜力

## 📊 评分机制

### 基础分数设定
- 可维护性：85分基础分
- 可读性：90分基础分
- 安全性：95分基础分
- 性能：88分基础分

### 扣分规则
| 问题类型 | 可维护性扣分 | 可读性扣分 | 安全性扣分 | 性能扣分 |
|---------|------------|----------|----------|--------|
| 错误(Error) | 3分/个 | - | 20分/个 | - |
| 警告(Warning) | 1分/个 | - | - | 8分/个 |
| 信息(Info) | 0.2分/个 | - | - | - |

### 特殊规则
- **可读性**：长行(-2分/个)、深层嵌套(-5分/个)、长函数(-3分/个)
- **安全性**：检测到密码/注入等关键词的问题额外-20分/个
- **注释奖励**：估算注释比例给予最高5分奖励

### 等级划分
- A级：≥90分 (优秀)
- B级：≥80分 (良好)  
- C级：≥70分 (合格)
- D级：≥60分 (需改进)
- F级：<60分 (不合格)

## 🧮 计算公式

```
总分 = 可维护性×0.3 + 可读性×0.25 + 安全性×0.25 + 性能×0.2
```

## 📈 输出指标

- 总分及等级
- 各维度详细得分
- 问题分布统计
- 每千行问题数(KLOC)
- 代码行数统计
